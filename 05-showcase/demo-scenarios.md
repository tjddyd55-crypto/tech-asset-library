# 데모 시나리오

## 시나리오 1: 신규 프로젝트 킥오프

- 목표: 1주 내 기본 구조 합의
- 사용 문서: `03-starter-kit/*`, `04-prompts/new-project-setup-prompt.md`
- 데모 포인트: 플랫폼 분리/권한 모델/기능 재사용 후보 선정

## 시나리오 2: 기존 기능 재사용

- 목표: 등급 A 기능을 신규 서비스에 빠르게 이식
- 사용 문서: `02-feature-library/feature-catalog.md`, `04-prompts/apply-feature-template.md`
- 데모 포인트: 코드 복사 없이 구조/계약 먼저 확정

## 시나리오 3: 난이도 높은 기능 리팩터링

- 목표: C 등급 기능을 B로 상향
- 사용 문서: `04-prompts/refactor-feature-template.md`
- 데모 포인트: 책임 분리, 권한/오류 경계 명확화, 재사용성 확보

## 시연 체크리스트

- 기능 템플릿 14개 항목이 모두 채워졌는가
- 플랫폼 구분이 명확한가
- 특정 프로젝트 종속 규칙이 표준으로 승격되지 않았는가
