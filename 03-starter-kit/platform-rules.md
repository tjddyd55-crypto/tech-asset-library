# 플랫폼 분리 규칙

## 플랫폼 구분

- PC 웹
- 모바일 웹
- 모바일 앱
- PC 설치형 앱
- 관리자 웹
- 고객용 앱/웹

## PC 웹

- 넓은 화면 기준으로 정보 밀도를 높인다.
- 사이드바, 상단바, 보조 패널 사용 가능.
- 테이블, 리스트, 상세 패널을 함께 배치할 수 있다.

## 모바일 웹

- 한 화면 한 목적을 원칙으로 한다.
- 카드형 리스트를 우선 사용한다.
- 상세는 별도 화면 또는 바텀시트로 분리한다.
- PC용 테이블을 그대로 사용하지 않는다.

## 모바일 앱

- 터치 영역을 충분히 확보한다.
- 카메라, 파일 첨부, 푸시 알림 가능성을 고려한다.
- 웹뷰인지 네이티브인지 초기에 결정한다.

## PC 설치형 앱

- Electron 등 설치형 런타임 전용 코드를 분리한다.
- 자동 업데이트를 고려한다.
- 타이틀바, 알림, 최소화/닫기 UX를 별도 관리한다.

## 공통 규칙

- 공통 로직은 공유하되 화면은 플랫폼별로 분리한다.
- 플랫폼별 진입점은 명확히 나눈다.
- 앱 전용 코드가 웹 빌드를 깨지 않도록 격리한다.
