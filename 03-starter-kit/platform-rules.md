# 플랫폼 분리 규칙

## 플랫폼 구분

- PC 웹
- 모바일 웹
- 네이티브 앱(iOS/Android)
- PC 설치형 클라이언트

## 공통 규칙

- 공통 도메인 모델은 공유 가능하지만 UI/입력 방식은 분리한다.
- 화면 레이아웃 규칙과 내비게이션 구조를 플랫폼별로 독립 관리한다.
- 성능/오프라인/디바이스 권한 요구사항을 플랫폼별로 문서화한다.

## 구현 가이드

- API 계약은 최대한 공통화하고, 클라이언트 어댑터에서 차이를 흡수한다.
- 파일 시스템, 카메라, 푸시, 생체인증 등 디바이스 의존 기능은 포트로 분리한다.
- 한 플랫폼에서의 임시 우회 코드를 다른 플랫폼 표준으로 확장하지 않는다.

## 검증 항목

- 플랫폼별 UX 시나리오 테스트를 분리했는가
- 플랫폼 종속 예외 처리를 명시했는가
- 배포/릴리즈 주기를 플랫폼별로 관리하는가
