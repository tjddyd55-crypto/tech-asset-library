# Figma 활용 워크플로우

## 목적

신규 프로젝트 스타터 생성 시 Figma를 활용하여 아키텍처, 사용자 흐름, UX, UI 시안을 시각화한다.

Figma는 코드 개발 전에 화면 구조와 이동 흐름을 검증하는 도구로 사용한다.

## Figma를 사용해야 하는 경우

- 프로젝트 구조를 한눈에 설명해야 할 때
- 관리자/고객/파트너 등 사용자별 흐름이 다를 때
- PC 웹과 모바일 웹 화면 구조를 분리해야 할 때
- 고객앱, 관리자웹, PC 설치형 앱 등 여러 플랫폼이 있을 때
- 투자자 또는 협력사에게 보여줄 시각 자료가 필요할 때
- Cursor가 화면을 만들기 전에 기준 UI가 필요할 때

## 권장 생성 순서

### 1단계: 서비스 구조도

목적:

- 서비스 전체 구조를 시각화한다.
- 사용자 역할과 주요 기능을 연결한다.

포함 내용:

- 사용자 유형
- 주요 메뉴
- 핵심 기능
- 데이터 흐름
- 외부 연동

### 2단계: 사용자 이동 흐름도

목적:

- 사용자가 어떤 순서로 화면을 이동하는지 확인한다.

포함 내용:

- 로그인 전/후 흐름
- 관리자 흐름
- 고객 흐름
- 파트너 흐름
- 파일 제출/승인/메시지 같은 핵심 업무 흐름

### 3단계: 정보 구조 IA

목적:

- 메뉴 구조를 먼저 정리한다.

포함 내용:

- PC 메뉴 구조
- 모바일 메뉴 구조
- 관리자 메뉴
- 고객앱 메뉴
- 권한별 노출 메뉴

### 4단계: 와이어프레임

목적:

- 실제 화면 배치 기준을 만든다.

포함 내용:

- PC 관리자 대시보드
- 모바일 홈
- 리스트 화면
- 상세 화면
- 작성/등록 화면
- 모달/바텀시트

### 5단계: 디자인 시스템 초안

목적:

- 개발 전에 색상, 버튼, 카드, 폼, 리스트 스타일을 통일한다.

포함 내용:

- 색상 토큰
- 폰트 크기
- 간격
- 버튼
- 카드
- 입력창
- 배지
- 모달
- 모바일 하단 메뉴

### 6단계: 클릭 프로토타입

목적:

- 사용자가 버튼을 눌렀을 때 어디로 이동하는지 검증한다.

포함 내용:

- 주요 버튼 이동
- 리스트에서 상세 이동
- 작성 후 완료 화면 이동
- 고객 제출 흐름
- 관리자 처리 흐름

## Cursor 개발과 연결하는 방식

Figma 산출물이 만들어지면 Cursor에는 다음 기준으로 지시한다.

```text
Figma 시안을 기준으로 구현한다.
단, Figma의 시각 구조는 참고하되 프로젝트의 기존 디자인 토큰과 컴포넌트 규칙을 우선한다.
PC와 모바일 화면은 분리한다.
공통 로직만 shared로 분리한다.
Figma에서 정의한 화면 이동 흐름을 라우트 구조에 반영한다.
```

## Figma 산출물 체크리스트

- 서비스 구조도 있음
- 사용자 흐름도 있음
- PC IA 있음
- 모바일 IA 있음
- PC 핵심 화면 있음
- 모바일 핵심 화면 있음
- 권한별 화면 차이 있음
- 디자인 시스템 초안 있음
- 클릭 흐름 있음
- Cursor 구현 지시문으로 변환 가능함

## 주의사항

- Figma 시안을 코드로 그대로 복사하지 않는다.
- 시각 디자인보다 플랫폼 구조와 UX 흐름을 먼저 확정한다.
- PC 화면을 모바일에 그대로 축소하지 않는다.
- 모바일 화면을 PC에 억지로 늘리지 않는다.
- Figma의 색상은 디자인 시스템 토큰으로 변환한다.
