# Figma 프롬프트 템플릿

신규 프로젝트의 구조와 UX를 Figma로 시각화할 때 사용하는 프롬프트 템플릿이다.

## 1. 서비스 구조도 프롬프트

```text
다음 신규 프로젝트의 서비스 구조도를 만들어줘.

프로젝트명:
[프로젝트명]

서비스 목적:
[서비스 목적]

주요 사용자:
[관리자, 담당자, 고객, 파트너 등]

필요 플랫폼:
[PC 웹, 모바일 웹, 모바일 앱, PC 설치형 앱 등]

주요 기능:
[기능 목록]

요구사항:
- 사용자 유형별로 접근 가능한 영역을 구분해줘.
- 핵심 데이터 흐름을 표시해줘.
- 관리자 영역과 고객 영역을 분리해줘.
- 추후 개발자가 보고 아키텍처를 이해할 수 있게 구성해줘.
- 너무 장식적인 디자인보다 구조 이해가 쉬운 다이어그램으로 만들어줘.
```

## 2. 사용자 흐름도 프롬프트

```text
다음 서비스의 사용자 이동 흐름도를 만들어줘.

프로젝트명:
[프로젝트명]

사용자 유형:
[사용자 유형]

핵심 업무 흐름:
[예: 고객 파일 제출, 담당자 검토, 메시지 발송, 승인 완료]

요구사항:
- 사용자별 시작 지점을 구분해줘.
- 버튼 클릭 후 이동하는 화면을 순서대로 보여줘.
- 성공/실패/보완 요청 같은 분기 흐름도 포함해줘.
- PC 관리자 흐름과 모바일 고객 흐름을 분리해줘.
- 개발자가 라우트 구조로 변환할 수 있게 화면 이름을 명확히 적어줘.
```

## 3. PC 관리자 화면 와이어프레임 프롬프트

```text
다음 서비스의 PC 관리자 화면 와이어프레임을 만들어줘.

프로젝트명:
[프로젝트명]

관리자 주요 업무:
[업무 목록]

필요 메뉴:
[메뉴 목록]

요구사항:
- 좌측 사이드바, 상단바, 메인 콘텐츠 구조를 사용해줘.
- 리스트와 상세를 함께 볼 수 있는 구조를 우선 고려해줘.
- 검색, 필터, 상태 배지, 주요 액션 버튼을 포함해줘.
- 정보가 많은 업무형 SaaS 화면처럼 구성해줘.
- 모바일 화면과 억지로 공유하지 않는 PC 전용 구조로 만들어줘.
```

## 4. 모바일 고객 화면 와이어프레임 프롬프트

```text
다음 서비스의 모바일 고객 화면 와이어프레임을 만들어줘.

프로젝트명:
[프로젝트명]

고객 주요 행동:
[예: 파일 제출, 메시지 확인, 상태 확인, 서명]

필요 화면:
[화면 목록]

요구사항:
- 한 화면 한 목적 원칙으로 구성해줘.
- 카드형 리스트와 큰 버튼을 사용해줘.
- 하단 탭 또는 간단한 홈 메뉴를 고려해줘.
- 고객이 헷갈리지 않도록 단계별 안내를 넣어줘.
- PC 관리자 화면을 축소한 형태가 아니라 모바일 전용 UX로 만들어줘.
```

## 5. 디자인 시스템 프롬프트

```text
다음 서비스에 사용할 디자인 시스템 초안을 만들어줘.

프로젝트명:
[프로젝트명]

서비스 분위기:
[밝은 SaaS, 업무형 다크모드, 고급 B2B, 심플 고객앱 등]

주요 사용자:
[사용자]

요구사항:
- 색상 토큰을 정의해줘.
- 버튼, 카드, 입력창, 모달, 리스트, 배지 스타일을 정의해줘.
- PC와 모바일 모두 사용할 수 있는 공통 디자인 토큰을 만들어줘.
- 단, PC와 모바일 레이아웃은 분리되는 것을 전제로 해줘.
- 개발자가 CSS 변수 또는 디자인 토큰으로 옮길 수 있게 정리해줘.
```

## 6. 클릭 프로토타입 프롬프트

```text
다음 서비스의 클릭 흐름 프로토타입을 구성해줘.

프로젝트명:
[프로젝트명]

핵심 플로우:
[예: 고객 파일 제출 -> 담당자 확인 -> 보완 요청 -> 고객 재제출 -> 완료]

요구사항:
- 각 버튼을 눌렀을 때 이동할 화면을 명확히 연결해줘.
- PC 관리자 흐름과 모바일 고객 흐름을 구분해줘.
- 완료, 실패, 보완 요청 같은 분기 상태를 포함해줘.
- 개발자가 이 흐름을 라우트와 상태값으로 변환할 수 있게 화면 이름과 상태명을 표시해줘.
```

## 사용 원칙

- Figma는 코드 개발 전에 화면 구조와 흐름을 검증하기 위해 사용한다.
- Figma 결과물은 Cursor 개발 지시문으로 변환되어야 한다.
- Figma 시안을 그대로 복사하지 않고, 디자인 시스템과 프로젝트 구조에 맞게 구현한다.
- 복잡한 기능은 먼저 흐름도와 와이어프레임을 만든 뒤 코드 작업을 시작한다.
