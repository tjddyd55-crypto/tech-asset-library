# 신규 프로젝트 스타터 생성기

## 목적

새 프로젝트를 시작하기 전에 필요한 정보를 입력하면, 프로젝트 아키텍처, 폴더 구조, 플랫폼 분리 기준, 권한 구조, 기술 스택, 기능 선택, 개발 순서, Cursor/Codex 지시문을 빠르게 생성하기 위한 문서 세트다.

## 왜 필요한가

기존 프로젝트 개발 과정에서 가장 많은 시간이 들어간 부분은 기능 자체보다 초기 구조 미정으로 인한 리팩터링이었다.

반복적으로 발생한 문제는 다음과 같다.

- PC 웹과 모바일 웹 구조가 섞임
- 모바일 앱과 웹 구조가 뒤늦게 분리됨
- PC 설치형 앱 구조가 나중에 붙으면서 코드가 꼬임
- 디자인 시스템이 늦게 정해져 화면 통일성이 떨어짐
- 권한 구조가 뒤늦게 바뀌어 API와 UI를 다시 수정함
- 파일관리, 메시지, 알림, 고객앱 같은 공통 기능을 매번 새로 설계함

이 문서 세트는 새 프로젝트 시작 시 이런 시행착오를 줄이는 것을 목표로 한다.

## 사용 흐름

1. `input-checklist.md`를 복사해 새 프로젝트 정보를 입력한다.
2. 입력 내용을 기준으로 `output-template.md` 형식의 결과물을 만든다.
3. 결과물에는 아키텍처, 폴더 구조, 권한 구조, 기술 스택, 기능 우선순위가 포함된다.
4. `cursor-execution-prompt.md`를 사용해 Cursor/Codex에 바로 개발 시작 지시를 준다.
5. 실제 개발 중 발견된 문제는 `generation-rules.md`에 반영해 스타터 기준을 개선한다.

## 생성되는 결과물

- 프로젝트 개요
- 플랫폼 구성
- 권한 구조
- 추천 기술 스택
- 추천 폴더 구조
- 디자인 시스템 방향
- 데이터/API 설계 초안
- 적용할 기술 자산 목록
- 1차 개발 순서
- Cursor/Codex 실행 지시문

## 핵심 원칙

- PC 웹, 모바일 웹, 모바일 앱, PC 설치형 앱은 처음부터 분리 여부를 결정한다.
- 화면은 플랫폼별로 분리하고 공통 로직만 공유한다.
- 디자인 시스템은 개발 초기에 먼저 고정한다.
- 권한 구조는 기능 개발 전에 먼저 정한다.
- 파일/메시지/알림/고객앱/서명/PDF 같은 기능은 기술자산보관소 기준으로 선택 적용한다.
- 특정 프로젝트의 구조를 무조건 표준으로 강제하지 않는다.
