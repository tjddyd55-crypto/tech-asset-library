# 권한 구조 규칙

## 기본 원칙

- 권한은 프론트 노출 제어와 백엔드 접근 제어를 모두 포함한다.
- 메뉴 노출과 데이터 접근 권한을 분리한다.
- 회사/팀/고객 연결 관계는 명확히 모델링한다.
- 역할명은 프로젝트별로 달라도 권한 패턴은 공통화한다.

## 권한 유형

- 슈퍼관리자
- 관리자
- 회사 관리자
- 팀장
- 팀원
- 일반 사용자
- 고객
- 외부 파트너
- 작성자
- 조회자

## 권한 설계 시 확인할 것

- 누가 생성할 수 있는가
- 누가 조회할 수 있는가
- 누가 수정할 수 있는가
- 누가 삭제할 수 있는가
- 누가 승인할 수 있는가
- 누가 위임할 수 있는가
- 데이터 범위가 개인/팀/회사/전체 중 어디까지인가

## 하지 말 것

- 프론트에서만 권한을 막지 않는다.
- 모든 권한을 단일 role 하나로만 처리하지 않는다.
- 고객 데이터와 내부 사용자 데이터를 같은 규칙으로 뭉뚱그리지 않는다.
- 테넌트/회사/팀 범위를 무시하지 않는다.
