# 권한 구조 규칙

## 기본 모델

- 사용자(User), 역할(Role), 권한(Permission), 스코프(Scope)를 분리한다.
- 읽기/쓰기/승인/관리 같은 행위를 권한 단위로 모델링한다.

## 설계 원칙

- 권한 판정은 UI와 API 모두에서 수행한다.
- 권한 없음 상태를 예외가 아닌 정상 시나리오로 처리한다.
- 관리자 권한은 최소 권한 원칙으로 분할한다.

## 감사와 추적

- 권한 변경은 반드시 이력(누가, 언제, 무엇을) 기록한다.
- 고위험 액션은 추가 승인 또는 재인증 정책을 둔다.
- 권한 실패 로그를 운영 지표로 수집한다.

## 금지 사항

- 하드코딩된 사용자 ID 예외 처리 금지
- 화면 숨김만으로 보안 처리 완료로 간주 금지
- 프로젝트별 임시 권한 플래그를 공통 규칙으로 승격 금지
