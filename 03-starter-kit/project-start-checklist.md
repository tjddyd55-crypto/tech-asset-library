# 신규 프로젝트 시작 체크리스트

## 1) 범위와 목표

- 문제 정의와 성공 지표를 문서로 고정했는가
- MVP 범위와 제외 범위를 구분했는가
- 플랫폼 범위(PC 웹/모바일 웹/앱/설치형)를 명시했는가

## 2) 아키텍처 분리

- 비즈니스 로직, 입출력, 인프라 계층을 분리했는가
- 도메인 모델과 API DTO를 분리했는가
- 변경 가능성이 높은 지점을 인터페이스로 분리했는가

## 3) 기능 인벤토리

- `01-project-inventory` 기준으로 후보 기능을 등록했는가
- 기능 등급(A/B/C/D)을 매겼는가
- 코드 복사 없이 목적/구조/적용 조건을 먼저 문서화했는가

## 4) 운영/보안

- 권한 모델(RBAC/ABAC)과 감사로그 전략을 정했는가
- 장애 대응(재시도/보상/알림) 기준을 정의했는가
- 개인정보/민감정보 저장 정책을 확정했는가

## 5) 품질 기준

- 테스트 전략(단위/통합/E2E)을 합의했는가
- 코드 리뷰 기준과 릴리즈 기준을 정의했는가
- 문서 업데이트 책임자를 지정했는가
