# 디자인 시스템 규칙

## 목적

- UI 품질 일관성과 개발 속도를 동시에 확보한다.
- 컴포넌트 단위 재사용을 기능 단위 재사용과 연결한다.

## 토큰 규칙

- 색상/간격/타이포를 토큰으로 선언하고 하드코딩을 금지한다.
- 플랫폼별 토큰 오버라이드는 허용하되 이름 체계는 유지한다.
- 토큰 변경 이력은 릴리즈 노트와 함께 관리한다.

## 컴포넌트 규칙

- 기본 컴포넌트(버튼/입력/모달/테이블)와 도메인 컴포넌트를 분리한다.
- 상태(기본/호버/비활성/오류)와 접근성 속성을 명시한다.
- 비즈니스 로직이 UI 컴포넌트 내부로 침투하지 않도록 한다.

## 문서화 규칙

- 각 컴포넌트에 사용 목적, 금지 패턴, 변형 규칙을 기록한다.
- 스크린샷만 저장하지 말고 동작/상태 규칙을 함께 기록한다.
