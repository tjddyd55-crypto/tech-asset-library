# 디자인 시스템 규칙

## 기본 원칙

- 색상, 간격, 폰트 크기, 테두리, 그림자는 토큰화한다.
- 컴포넌트에서 임의 색상 하드코딩을 금지한다.
- PC와 모바일은 같은 디자인 토큰을 쓰되 레이아웃은 분리한다.
- 버튼, 입력창, 카드, 모달, 리스트, 배지는 공통 스타일 기준을 따른다.

## 필수 토큰

- Primary
- Background
- Surface
- Border
- Text Main
- Text Sub
- Danger
- Success
- Warning
- Info

## 컴포넌트 기준

- Button
- Input

- Modal
- Badge

- List
- BottomSheet
- Sidebar
- Header
- EmptyState
- LoadingState

## 모바일 UX 기준

- 터치 영역은 작게 만들지 않는다.
- 주요 액션은 하단 또는 엄지 접근 영역에 둔다.
- 좁은 화면에서 과도한 정보 노출을 피한다.
- PC용 테이블 대신 카드/요약 리스트를 사용한다.

## PC UX 기준

- 좌측 메뉴 폭을 과도하게 넓히지 않는다.
- 리스트와 상세를 함께 볼 수 있게 구성한다.
- 반복 작업이 빠르게 가능하도록 버튼 위치를 일정하게 유지한다.
