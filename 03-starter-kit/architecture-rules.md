# 아키텍처 규칙

## 기본 원칙

- 화면 코드와 비즈니스 로직을 섞지 않는다.
- PC와 모바일 화면 컴포넌트는 처음부터 분리한다.
- 공통 로직, 타입, API 클라이언트, 디자인 토큰은 shared 영역에 둔다.
- 플랫폼 전용 코드는 플랫폼별 폴더에 격리한다.
- 기능 내부에 저장소/권한/라우트 구조를 고정하지 않는다.
- 권한 체크는 프론트와 백엔드 양쪽에서 수행한다.
- 파일 업로드/다운로드는 저장소 어댑터 방식으로 분리한다.

## 권장 계층

- apps: 실행 앱
- packages: 공통 패키지
- shared: 공통 타입/유틸/API
- features: 기능 모듈
- server: 백엔드/API/DB
- docs: 문서

## 하지 말 것

- PC/모바일을 하나의 거대한 반응형 컴포넌트로 억지 통합하지 않는다.
- Electron/Expo 전용 코드를 웹 공통 영역에 섞지 않는다.
- DB 테이블명, 라우트명, 권한명을 기능 내부에 하드코딩하지 않는다.
- 디자인 색상과 간격을 임의로 하드코딩하지 않는다.
- 기존 프로젝트 전체를 새 프로젝트에 통째로 복사하지 않는다.
