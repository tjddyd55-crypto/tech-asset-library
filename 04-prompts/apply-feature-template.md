- 원본 프로젝트:
- 원본 코드 위치:

적용 원칙:
1. 원본 기능의 목적과 UX는 유지한다.
2. 저장소, API 경로, DB 테이블명, 권한명은 현재 프로젝트 기준으로 연결한다.
3. 기능 내부에 특정 프로젝트 고정값을 넣지 않는다.

5. 공통 로직은 shared 또는 feature 내부 service로 분리한다.
6. 기존 기능을 깨지 않도록 변경 범위를 최소화한다.

필요 작업:
- 프론트 UI 적용
- API 연결
- DB/스토리지 연결
- 권한 체크
- 오류 처리
- 빈 상태/로딩 상태 처리
- 모바일 UX 확인
- PC UX 확인
