# 기능 리팩터링 지시문 템플릿

```text
목표:
[기능명] 기능을 재사용 가능한 표준 기능 자산으로 리팩터링한다.

현재 문제:
- 프로젝트 고정값이 섞여 있음
- 저장소/권한/API가 기능 내부에 박혀 있음
- PC/모바일 화면이 섞여 있음
- 오류 처리/빈 상태가 부족함
- 코드 위치가 흩어져 있음

리팩터링 원칙:
1. 원본 기능의 동작은 유지한다.
2. 도메인 고정값을 제거한다.
3. 저장소와 API는 외부에서 연결 가능하게 분리한다.
4. UI와 비즈니스 로직을 분리한다.
5. PC/모바일 UI는 필요한 경우 분리한다.
6. 공통 타입과 유틸을 정리한다.
7. 테스트 또는 수동 검증 기준을 문서화한다.

결과물:
- README.md
- source-location.md
- clean-version/
- dependencies.md
- apply-prompt.md
- refactor-notes.md

절대 하지 말 것:
- 원본 프로젝트 코드를 통째로 복사하지 말 것
- 동작 확인 없이 clean-version으로 저장하지 말 것
- 특정 프로젝트 전용 구조를 표준이라고 기록하지 말 것
```
