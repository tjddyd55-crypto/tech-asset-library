# Cursor Master Prompt

```text
이 프로젝트는 기술자산보관소 기준으로 개발한다.

절대 원칙:
1. PC와 모바일 화면을 억지로 하나로 합치지 않는다.
2. 공통 로직은 shared에 두고 화면 컴포넌트는 플랫폼별로 분리한다.
3. 기능 내부에 특정 프로젝트의 저장소, 권한, 라우트, DB 테이블명을 고정하지 않는다.
4. 디자인 색상, 간격, 폰트 크기는 디자인 시스템 토큰을 따른다.
5. 권한 체크는 프론트와 백엔드 양쪽에서 수행한다.
6. 기존 프로젝트 코드를 통째로 복사하지 않는다.
7. 필요한 기능은 기능 자산 문서를 기준으로 최소 단위로 적용한다.

작업 방식:
- 먼저 현재 구조를 파악한다.
- 변경 범위를 작게 잡는다.
- 기존 기능 회귀를 막는다.
- 작업 후 변경 파일과 검증 결과를 요약한다.
```
