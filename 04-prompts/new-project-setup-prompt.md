# 신규 프로젝트 생성 지시문 템플릿

```text
새 프로젝트를 생성한다.

프로젝트명:
[프로젝트명 입력]

프로젝트 목적:
[프로젝트 목적 입력]

대상 사용자:
[관리자/직원/고객/파트너 등 입력]

플랫폼:
- PC 웹:
- 모바일 웹:
- Android 앱:
- iOS 앱:
- PC 설치형 앱:

기본 아키텍처:
1. PC와 모바일 UI는 처음부터 분리한다.
2. 공통 로직, 타입, API 클라이언트, 디자인 토큰은 shared에 둔다.
3. 플랫폼 전용 코드는 플랫폼별 폴더에 격리한다.
4. 권한 체크는 프론트와 백엔드 양쪽에서 수행한다.
5. 파일 저장소는 어댑터 방식으로 설계한다.
6. 기능 내부에 특정 프로젝트 고정값을 넣지 않는다.

적용할 기능 자산:
- [기능명 1]
- [기능명 2]
- [기능명 3]

절대 하지 말 것:
- PC/모바일을 하나의 반응형 컴포넌트로 억지 통합하지 말 것
- 디자인 색상을 임의로 하드코딩하지 말 것
- 기존 프로젝트 전체를 통째로 복사하지 말 것
- 권한 체크를 프론트에만 두지 말 것

완료 후:
- 생성된 구조를 요약한다.
- 선택한 기능 자산과 적용 위치를 정리한다.
- 다음 개발 순서를 제안한다.
```
