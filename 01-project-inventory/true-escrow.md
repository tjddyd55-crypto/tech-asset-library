## 후보 기능

### 1) 거래 단계별 자금 잠금/해제 흐름 (등급: B)

- 기능명: 거래 단계별 자금 잠금/해제 흐름
- 원본 프로젝트: true-escrow
- 원본 코드 위치: `미기재 (추후 경로 등록)`
- 기능 목적: 거래 신뢰 확보를 위해 상태 기반 자금 이동 제어
- 재사용 가능 분야: 마켓플레이스, B2B 정산, 예약결제
- 필요한 프론트 구조: 거래 타임라인, 정산 상태 카드
- 필요한 백엔드/API 구조: 거래 상태 API, 정산 지시 API, 감사로그 API
- 필요한 DB/스토리지 구조: 거래, 정산, 상태이력, 분쟁이력 테이블
- 필요한 권한 구조: 구매자/판매자/중재자/운영자 권한 분리
- 외부 의존성: PG/정산 파트너 API
- 재사용 난이도: 중간
- 정리 필요 여부: 예 (외부 PG 어댑터 추상화 필요)
- 새 프로젝트 적용 지시문: 거래 상태머신과 정산 어댑터를 분리 구현
- 투자/소개용 설명 문구: 거래 신뢰성과 분쟁 대응력을 동시에 확보하는 핵심 엔진

### 2) 분쟁 처리 워크플로우 (등급: C)

- 기능명: 분쟁 처리 워크플로우
- 원본 프로젝트: true-escrow
- 원본 코드 위치: `미기재 (추후 경로 등록)`
- 기능 목적: 이의 제기부터 중재 판정까지 표준 프로세스 제공
- 재사용 가능 분야: 커머스 CS, 계약 분쟁 관리
- 필요한 프론트 구조: 분쟁 접수 폼, 증빙 업로드, 판정 이력
- 필요한 백엔드/API 구조: 분쟁 생성/상태변경 API, 증빙 관리 API
- 필요한 DB/스토리지 구조: 분쟁, 증빙메타, 판정로그 테이블
- 필요한 권한 구조: 당사자 열람 제한 + 중재자 승인 권한
- 외부 의존성: 파일 저장소, 알림 채널
- 재사용 난이도: 높음
- 정리 필요 여부: 예 (증빙 스키마 통일 필요)
- 새 프로젝트 적용 지시문: 분쟁 상태 전이 규칙을 도메인 정책으로 독립
- 투자/소개용 설명 문구: 플랫폼 리스크를 낮추는 사후 신뢰 보장 체계
