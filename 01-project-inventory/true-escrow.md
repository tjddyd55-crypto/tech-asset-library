# True Escrow 프로젝트 인벤토리

## 프로젝트 개요

거래 과정에서 단계별 조건을 설정하고, 각 단계의 이행 여부에 따라 대금 지급을 관리하는 블록체인 기반 에스크로/증명 서비스다.

## 설명

계약 당사자가 약속한 행위를 일정에 맞게 수행하고, 고객이 승인하면 계약된 자금이 일부 또는 전체 지급되는 구조를 목표로 한다. 자료는 블록체인 또는 변경 불가능한 기록 구조에 저장하여 분쟁 시 증명 자료로 활용한다.

## 사용자 유형

- 의뢰자
- 수행자
- 검수자
- 관리자
- 분쟁 조정자

## 주요 기능 후보

- 거래 계약 생성
- 단계별 타임라인
- 조건부 지급
- 산출물 업로드
- 고객 승인
- 분쟁 대응
- 블록체인 기록
- 콘텐츠 원본 증명
- 변경 불가능한 해시 기록
- 증명서 발급

## 재사용 가능한 기능 후보

| 기능 | 분류 | 등급 초안 | 비고 |
|---|---|---|---|
| 단계별 계약 타임라인 | 워크플로우 | B | 프로젝트/납품/검수 서비스에 재사용 가능 |
| 조건부 지급 | 거래/정산 | C | 결제/블록체인 의존성 확인 필요 |
| 산출물 업로드 | 문서/증명 | B | 파일관리와 연계 가능 |
| 고객 승인 플로우 | 승인/검수 | B | ERP/업무관리에도 재사용 가능 |
| 콘텐츠 원본 증명 | 블록체인/증명 | C | 별도 표준화 필요 |
| 분쟁 대응 기록 | 감사로그 | B | 이력/감사 기능으로 재사용 가능 |

## 스타터킷에 반영할 구조

- 타임라인 기반 업무 플로우를 범용 기능으로 분리한다.
- 승인/검수/지급 조건을 도메인별로 교체 가능하게 설계한다.
- 증명 기능은 블록체인 의존성과 일반 DB 기록을 분리한다.
- 파일 업로드와 해시/감사로그를 연결하는 구조를 검토한다.

## 우선 표준화할 기능 TOP 5

1. 단계별 타임라인
2. 승인/검수 플로우
3. 산출물 업로드
4. 감사로그/이력 기록
5. 콘텐츠 원본 증명
