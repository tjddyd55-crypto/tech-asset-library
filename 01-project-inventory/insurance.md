# Insurance 프로젝트 인벤토리

## 프로젝트 개요

보험 FC/GA 업무 자동화를 목적으로 한 업무형 SaaS 프로젝트다.

## 설명

FC/GA/고객앱/청구/소식지/파일관리/일정관리 기능을 포함한다.

## 사용자 유형

- 슈퍼관리자
- 관리자
- GA/회사 관리자
- 팀장
- FC/설계사
- 고객
- 원수사
- 손해사정사

## 주요 기능 후보

- 고객 관리
- 고객앱 연결 코드
- 고객 청구 파일 접수
- 전체 소식지
- 개별 소식지
- 개인 메시지
- 손사인
- PDF 좌표 기반 자동화
- R2/S3 파일관리
- 폴더 트리 파일관리
- 팀 관리
- 팀 자료실
- 일정관리
- 권한별 게시판
- 엑셀 업로드/컬럼 매칭
- PC/모바일 분리 구조
- Electron PC 설치형 앱
- 자동 업데이트

## 재사용 가능한 기능 후보

| 기능 | 분류 | 등급 초안 | 비고 |
|---|---|---|---|
| 손사인 | 문서/계약 | A | 독립 컴포넌트화 가능 |
| PDF 좌표 자동화 | 문서/계약 | B | DB/API/파일 구조 의존성 확인 필요 |
| 고객 청구 파일 접수 | 고객앱 | B | 고객 연결/파일 업로드/권한 연결 필요 |
| 개인 메시지 | 메시지 | B | 고객앱/관리자 화면 연결 필요 |
| R2/S3 파일관리 | 스토리지 | B | 용량/권한/삭제 정책 포함 |
| 폴더 트리 파일관리 | 스토리지/UI | A/B | UI와 API 분리 필요 |
| PC/모바일 분리 구조 | 아키텍처 | C | 코드보다 규칙 자산으로 관리 |
| Electron 자동 업데이트 | 운영 | B | PC 설치형 스타터에 반영 |
| 엑셀 업로드/컬럼 매칭 | 데이터 | B | 업종별 매핑 규칙 분리 필요 |
| 팀/일정관리 | 조직/업무 | B | 권한 구조 확인 필요 |

## 스타터킷에 반영할 구조

- PC 웹과 모바일 웹은 화면 컴포넌트를 처음부터 분리한다.
- 공통 로직, API 클라이언트, 타입, 디자인 토큰은 shared 영역에 둔다.
- Electron 전용 코드는 PC 설치형 앱 영역에 격리한다.
- 파일 업로드 기능은 저장소를 고정하지 않고 어댑터 방식으로 설계한다.
- 권한 체크는 프론트와 백엔드 양쪽에 둔다.
- 고객앱 기능은 관리자 기능과 별도 라우트/레이아웃으로 분리한다.
