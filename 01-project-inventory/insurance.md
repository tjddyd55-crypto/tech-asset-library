## 후보 기능

### 1) 심사 단계 워크플로우 엔진 (등급: B)

- 기능명: 심사 단계 워크플로우 엔진
- 원본 프로젝트: insurance
- 원본 코드 위치: `미기재 (추후 경로 등록)`
- 기능 목적: 심사 단계를 정책 기반으로 분리해 운영자가 규칙 변경 가능하도록 지원
- 재사용 가능 분야: 보험/대출/승인형 업무 시스템
- 필요한 프론트 구조: 단계 상태 타임라인, 심사 코멘트 패널
- 필요한 백엔드/API 구조: 상태 전이 API, 정책 조회 API, 감사로그 API
- 필요한 DB/스토리지 구조: 심사건, 상태이력, 정책버전 테이블
- 필요한 권한 구조: 심사자/검수자/관리자 역할 분리
- 외부 의존성: 없음(내부 정책 엔진 가정)
- 재사용 난이도: 중간
- 정리 필요 여부: 예 (정책 로딩부 모듈화 필요)
- 새 프로젝트 적용 지시문: 상태 전이 규칙을 도메인 정책으로 분리 후 API 계약부터 고정
- 투자/소개용 설명 문구: 운영 정책 변경 비용을 크게 줄이는 심사 자동화 기반 기능

### 2) 약관/동의 버전 관리 (등급: A)

- 기능명: 약관/동의 버전 관리
- 원본 프로젝트: insurance
- 원본 코드 위치: `미기재 (추후 경로 등록)`
- 기능 목적: 가입 시점별 동의 이력을 버전으로 추적
- 재사용 가능 분야: 핀테크, 커머스, 헬스케어
- 필요한 프론트 구조: 약관 버전 목록, 변경 요약 UI
- 필요한 백엔드/API 구조: 최신 약관 조회, 동의 저장, 동의 이력 조회 API
- 필요한 DB/스토리지 구조: 약관버전, 사용자동의이력 테이블
- 필요한 권한 구조: 약관 작성 권한과 배포 승인 권한 분리
- 외부 의존성: 전자서명 모듈(선택)
- 재사용 난이도: 낮음
- 정리 필요 여부: 아니오
- 새 프로젝트 적용 지시문: 가입 플로우 시작 전에 약관 버전 조회 API를 선호출
- 투자/소개용 설명 문구: 규제 대응과 사용자 신뢰를 동시에 확보하는 동의 추적 기능
