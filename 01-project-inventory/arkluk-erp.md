# Arkluk ERP 프로젝트 인벤토리

## 프로젝트 개요

일반 소비자 판매가 아닌 판매자와 구매자의 거래 편의를 위한 B2B 거래 ERP 시스템이다.

## 설명

건축자재회사 등 판매자가 상품을 등록하고, 지정된 대리점 또는 구매자가 주문, 결제, 문의를 처리하는 거래 시스템이다.

## 사용자 유형

- 관리자
- 판매자
- 구매자
- 대리점
- 담당자
- 정산/회계 담당자

## 주요 기능 후보

- 상품 등록
- 판매자/구매자 권한 구조
- 주문 생성
- 주문 배정
- 결제/정산
- 문의
- 인보이스
- 이메일 자동화
- 주문 모니터링
- 주문 관련 cron
- 엑셀 처리
- 파일 업로드

## 재사용 가능한 기능 후보

| 기능 | 분류 | 등급 초안 | 비고 |
|---|---|---|---|
| B2B 상품 등록 | ERP/거래 | B | 카탈로그형 서비스에 재사용 가능 |
| 주문/발주 | ERP/거래 | B | 거래/재고/정산 확장 가능 |
| 인보이스 | 문서/정산 | B | PDF/메일 연계 가능 |
| 메일 자동화 | 운영 | B | 알림/청구/주문에 재사용 가능 |
| 주문 모니터링 cron | 운영/자동화 | B | 배치 작업 표준화 가능 |
| 판매자/구매자 권한 | 권한 | B | B2B SaaS 스타터에 반영 |

## 스타터킷에 반영할 구조

- 판매자/구매자 권한을 기본 B2B 권한 패턴으로 정리한다.
- 주문/인보이스/메일/모니터링은 ERP 기능 묶음으로 관리한다.
- cron 작업은 서버 운영 자산으로 별도 분류한다.
- Prisma 기반 DB 설계 패턴을 Next.js 스타터 후보로 관리한다.

## 우선 표준화할 기능 TOP 5

1. 판매자/구매자 권한 구조
2. 상품 등록
3. 주문/발주
4. 인보이스
5. 메일/cron 자동화
