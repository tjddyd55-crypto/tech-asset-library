## 프로젝트 메모

- 도메인: 내부 운영(재고/구매/정산/권한) 통합 관리
- 현재 상태: 모듈형 기능 추출 가능성 높음

## 후보 기능

### 1) 멀티 롤 기반 메뉴/권한 매트릭스 (등급: A)

- 기능명: 멀티 롤 기반 메뉴/권한 매트릭스
- 원본 프로젝트: arkluk-erp
- 원본 코드 위치: `미기재 (추후 경로 등록)`
- 기능 목적: 역할 조합별 메뉴 노출/행위 권한 통합 제어
- 재사용 가능 분야: 관리자 페이지, 백오피스 전반
- 필요한 프론트 구조: 메뉴 가드, 액션 버튼 가드, 권한 디버그 패널
- 필요한 백엔드/API 구조: 권한 매트릭스 API, 사용자 권한 조회 API
- 필요한 DB/스토리지 구조: 역할, 권한, 역할-권한 매핑 테이블
- 필요한 권한 구조: 슈퍼관리자 변경 이력 강제
- 외부 의존성: SSO(선택)
- 재사용 난이도: 낮음
- 정리 필요 여부: 아니오
- 새 프로젝트 적용 지시문: 라우트/액션 단위 권한 키를 먼저 표준화
- 투자/소개용 설명 문구: 운영 보안성과 기능 확장성을 동시에 만족시키는 권한 기반

### 2) 재고 이동 이력 추적 (등급: B)

- 기능명: 재고 이동 이력 추적
- 원본 프로젝트: arkluk-erp
- 원본 코드 위치: `미기재 (추후 경로 등록)`
- 기능 목적: 입출고/이동/조정 이벤트를 이벤트 로그로 일관 기록
- 재사용 가능 분야: 물류, 자산관리, 장비관리
- 필요한 프론트 구조: 재고 타임라인, 필터/검색 UI
- 필요한 백엔드/API 구조: 이동 기록 API, 배치 동기화 API
- 필요한 DB/스토리지 구조: 재고스냅샷 + 이벤트로그 이중 구조
- 필요한 권한 구조: 승인형 이동 처리 권한
- 외부 의존성: 바코드/스캐너 연동(선택)
- 재사용 난이도: 중간
- 정리 필요 여부: 예 (이벤트 명세 통일 필요)
- 새 프로젝트 적용 지시문: 스냅샷 모델과 이벤트 모델을 분리 저장
- 투자/소개용 설명 문구: 운영 손실을 줄이고 감사 대응을 강화하는 추적 기능
