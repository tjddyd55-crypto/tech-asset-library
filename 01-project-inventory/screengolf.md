# Screengolf 프로젝트 인벤토리

## 프로젝트 개요

스크린골프 연습 시 휘발성으로 사라지는 샷 데이터를 수집하고, 기록/분석을 통해 사용자가 샷 관리를 할 수 있게 하는 프로젝트다.

## 설명

샷데이터 API 값을 받지 못하는 상황에서 화면 데이터를 읽어 수집하는 구조를 목표로 한다. 기록 프로그램이 없는 스크린연습장 도입을 목적으로 한다.

## 사용자 유형

- 일반 사용자
- 연습장 관리자
- 코치
- 데이터 관리자

## 주요 기능 후보

- 화면 기반 데이터 수집
- 샷 데이터 기록
- 사용자별 기록 관리
- 샷 분석
- 통계
- 랭킹
- 기록 히스토리
- 외부 API 없는 데이터 추출

## 재사용 가능한 기능 후보

| 기능 | 분류 | 등급 초안 | 비고 |
|---|---|---|---|
| 화면 기반 데이터 수집 | 데이터 수집 | C | OCR/크롤링/화면 인식 방식 검토 필요 |
| 기록 저장 | 데이터 | B | 사용자별 히스토리 기능으로 재사용 |
| 통계/분석 | 분석 | B | 다른 기록형 서비스에 확장 가능 |
| 랭킹 | 커뮤니티/통계 | B | 스포츠/교육/성과관리 서비스에 재사용 |
| 외부 API 없는 추출 구조 | 수집/자동화 | C | 안정화 필요 |

## 스타터킷에 반영할 구조

- 데이터 수집과 분석 로직을 분리한다.
- 외부 API가 없는 환경에서 화면/문서/HTML을 읽어 구조화하는 기능 자산으로 분류한다.
- 기록형 서비스의 사용자별 히스토리/통계/랭킹 패턴을 정리한다.

## 우선 표준화할 기능 TOP 5

1. 기록 저장 구조
2. 사용자별 히스토리
3. 통계/분석
4. 랭킹
5. 화면 기반 데이터 수집 방식
