## 프로젝트 메모

- 도메인: 스크린 스포츠 예약/운영/정산
- 현재 상태: 사용자 경험 + 운영 자동화 기능 추출 대상

## 후보 기능

### 1) 타임슬롯 예약 충돌 방지 (등급: A)

- 기능명: 타임슬롯 예약 충돌 방지
- 원본 프로젝트: screengolf
- 원본 코드 위치: `미기재 (추후 경로 등록)`
- 기능 목적: 동일 자원/시간대 중복 예약을 원자적으로 차단
- 재사용 가능 분야: 예약 서비스 전반(레슨, 공유오피스, 병원)
- 필요한 프론트 구조: 슬롯 선택 UI, 충돌 피드백 모달
- 필요한 백엔드/API 구조: 예약 생성 API(트랜잭션), 슬롯 잠금 API
- 필요한 DB/스토리지 구조: 슬롯 인덱스, 예약 테이블, 잠금 기록
- 필요한 권한 구조: 지점 운영자 대리 예약 권한
- 외부 의존성: 결제 모듈(예약금 사용 시)
- 재사용 난이도: 낮음
- 정리 필요 여부: 아니오
- 새 프로젝트 적용 지시문: DB 유니크 제약 + 애플리케이션 잠금을 함께 적용
- 투자/소개용 설명 문구: 예약 실패와 고객 불만을 줄이는 운영 안정성 핵심 기능

### 2) 지점별 운영 대시보드 (등급: B)

- 기능명: 지점별 운영 대시보드
- 원본 프로젝트: screengolf
- 원본 코드 위치: `미기재 (추후 경로 등록)`
- 기능 목적: 예약률/매출/노쇼율 등 운영 KPI를 지점 단위로 제공
- 재사용 가능 분야: 프랜차이즈 운영, 멀티 브랜치 서비스
- 필요한 프론트 구조: KPI 카드, 기간 필터, 추세 차트
- 필요한 백엔드/API 구조: 지표 집계 API, 캐시 갱신 배치
- 필요한 DB/스토리지 구조: 집계 테이블, 캐시 저장소
- 필요한 권한 구조: 본사/지점 권한 스코프 분리
- 외부 의존성: BI 연동(선택)
- 재사용 난이도: 중간
- 정리 필요 여부: 예 (지표 산식 문서화 필요)
- 새 프로젝트 적용 지시문: 지표 정의서 확정 후 API 스키마를 고정
- 투자/소개용 설명 문구: 운영 데이터를 실시간 의사결정으로 전환하는 대시보드 자산
