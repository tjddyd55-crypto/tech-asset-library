## 후보 기능

### 1) 지원서 단계별 검토 보드 (등급: B)

- 기능명: 지원서 단계별 검토 보드
- 원본 프로젝트: global-audition
- 원본 코드 위치: `미기재 (추후 경로 등록)`
- 기능 목적: 접수-1차-2차-합격 단계 진행을 시각적으로 관리
- 재사용 가능 분야: 채용 ATS, 파이프라인 영업 시스템
- 필요한 프론트 구조: 칸반 보드, 단계 이동 드래그 인터랙션
- 필요한 백엔드/API 구조: 단계 이동 API, 검토 코멘트 API
- 필요한 DB/스토리지 구조: 지원서, 단계로그, 코멘트 테이블
- 필요한 권한 구조: 심사위원/매니저/운영자 권한 분리
- 외부 의존성: 파일 저장소(포트폴리오 업로드)
- 재사용 난이도: 중간
- 정리 필요 여부: 예 (드래그 이벤트-API 결합 분리 필요)
- 새 프로젝트 적용 지시문: 단계 정의를 구성값으로 분리해 도메인별 재사용
- 투자/소개용 설명 문구: 선발 프로세스를 빠르게 가시화해 운영 효율을 높이는 기능

### 2) 심사 점수 집계/가중치 시스템 (등급: C)

- 기능명: 심사 점수 집계/가중치 시스템
- 원본 프로젝트: global-audition
- 원본 코드 위치: `미기재 (추후 경로 등록)`
- 기능 목적: 다중 심사자의 점수를 가중치로 통합
- 재사용 가능 분야: 평가 시스템, 설문 분석
- 필요한 프론트 구조: 평가표 입력 UI, 점수 분포 차트
- 필요한 백엔드/API 구조: 점수 저장 API, 집계 API, 이력 API
- 필요한 DB/스토리지 구조: 평가항목, 점수기록, 집계결과 테이블
- 필요한 권한 구조: 심사자 입력 권한/운영자 조정 권한 분리
- 외부 의존성: 통계 라이브러리(선택)
- 재사용 난이도: 높음
- 정리 필요 여부: 예 (점수 정책 버전 관리 필요)
- 새 프로젝트 적용 지시문: 평가 정책 버전 엔터티를 먼저 설계 후 집계 로직 연결
- 투자/소개용 설명 문구: 주관적 평가를 데이터 기반으로 표준화하는 핵심 기능
