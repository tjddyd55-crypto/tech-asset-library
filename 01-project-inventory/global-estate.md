## 후보 기능

### 1) 매물 상태 수명주기 관리 (등급: A)

- 기능명: 매물 상태 수명주기 관리
- 원본 프로젝트: global-estate
- 원본 코드 위치: `미기재 (추후 경로 등록)`
- 기능 목적: 등록-검수-공개-보류-종료 상태를 일관되게 제어
- 재사용 가능 분야: 커머스 상품, 채용 공고, 중고거래
- 필요한 프론트 구조: 상태 배지, 상태 변경 액션 패널
- 필요한 백엔드/API 구조: 상태 변경 API, 상태 이력 조회 API
- 필요한 DB/스토리지 구조: 객체 본문 + 상태이력 분리 테이블
- 필요한 권한 구조: 등록자/검수자/운영자 역할 분리
- 외부 의존성: 메시지 큐(알림 동기화 시)
- 재사용 난이도: 낮음
- 정리 필요 여부: 아니오
- 새 프로젝트 적용 지시문: 상태 전이를 enum 하드코딩이 아닌 정책 객체로 분리
- 투자/소개용 설명 문구: 거래 객체의 신뢰성과 운영 추적성을 동시에 확보하는 핵심 엔진

### 2) 매물 노출 스코어링 (등급: C)

- 기능명: 매물 노출 스코어링
- 원본 프로젝트: global-estate
- 원본 코드 위치: `미기재 (추후 경로 등록)`
- 기능 목적: 매물 품질/신선도/신뢰 지표로 노출 순서를 조정
- 재사용 가능 분야: 콘텐츠 피드, 상품 랭킹
- 필요한 프론트 구조: 추천 영역, 스코어 설명 툴팁
- 필요한 백엔드/API 구조: 스코어 계산 배치 + 조회 API
- 필요한 DB/스토리지 구조: 지표 집계 테이블, 캐시 레이어
- 필요한 권한 구조: 운영 정책 편집 권한
- 외부 의존성: 검색엔진/캐시 서버
- 재사용 난이도: 높음
- 정리 필요 여부: 예 (지표 가중치 구성 외부화 필요)
- 새 프로젝트 적용 지시문: 가중치를 코드에서 분리해 운영 정책으로 관리
- 투자/소개용 설명 문구: 사용자 전환율 향상을 위한 품질 기반 노출 최적화 기능
