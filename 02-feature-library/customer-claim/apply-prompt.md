# 고객 청구 파일 접수 기능 적용 지시문

```text
목표:
현재 프로젝트에 고객 파일 접수 기능을 적용한다.

기능 목적:
고객이 모바일 웹 또는 고객앱에서 파일을 직접 제출하고, 담당자 또는 관리자가 접수 내역을 확인하여 업무 처리를 진행할 수 있게 한다.

적용 원칙:
1. 고객용 화면과 내부 담당자 화면을 분리한다.
2. 파일 업로드는 storage-r2 기능 자산과 연결한다.
3. 고객 연결은 링크, 토큰, 코드 등 프로젝트에 맞는 방식으로 처리한다.
4. 접수 상태는 업무에 맞게 확장 가능하게 만든다.
5. 고객에게 노출되는 정보와 내부 담당자 정보는 분리한다.
6. 담당자 코멘트 또는 추가 요청은 메시지 기능과 연결 가능하게 만든다.
7. 모바일 파일 첨부 UX를 우선 고려한다.

필수 고객 UX:
- 접속 링크 진입
- 본인 또는 연결 정보 확인
- 파일 첨부
- 제출 완료 안내
- 접수 상태 확인
- 추가 요청 확인

필수 담당자 UX:
- 접수 목록
- 접수 상세
- 제출 파일 확인
- 상태 변경
- 추가 요청 작성
- 처리 완료 표시

필수 서버 기능:
- 고객 연결 확인
- 접수 생성
- 파일 연결
- 접수 목록 조회
- 접수 상세 조회
- 상태 변경
- 권한 확인

데이터 연결:
- customerId 또는 연결 대상 ID를 외부에서 주입한다.
- 담당자/회사/팀 범위는 프로젝트 권한 구조에 맞춘다.
- 파일은 fileId 기준으로 접수 내역에 연결한다.

절대 하지 말 것:
- 고객용 화면에 내부 관리자 전용 정보를 노출하지 말 것
- 파일을 접수 내역과 연결하지 않고 단독 저장하지 말 것
- 고객이 다른 고객의 접수 내역을 볼 수 있게 하지 말 것
- PC용 테이블 화면을 모바일 고객용 화면에 그대로 사용하지 말 것
- 보험 청구 전용 필드를 모든 프로젝트 표준으로 강제하지 말 것

완료 후 보고:
- 생성/수정한 파일 목록
- 고객 제출 흐름
- 담당자 확인 흐름
- 파일 연결 방식
- 상태 관리 방식
- 남은 TODO
```
