# 고객 청구 파일 접수 기능 의존성

## 관련 기능 자산

- storage-r2: 파일 업로드와 파일 메타데이터 관리
- personal-message: 고객에게 추가 요청 또는 안내 메시지 발송
- signature: 서명이 필요한 서류 접수 시 연결 가능
- pdf-coordinate: 제출된 서류 또는 신청서 자동 작성 시 연결 가능

## 프론트 구성 요소

### 고객용 화면

- 접속 링크 진입 화면
- 고객 식별 또는 연결 확인 화면
- 파일 제출 화면
- 제출 상태 확인 화면
- 추가 요청 확인 화면
- 모바일 카드형 UI

### 담당자/관리자 화면

- 접수 목록
- 접수 상세
- 제출 파일 목록
- 상태 변경 액션
- 추가 요청 작성
- 처리 완료 표시

## 서버 구성 요소

- 고객 연결 확인
- 접수 내역 생성
- 파일 메타데이터 연결
- 접수 상태 변경
- 담당자 권한 확인
- 고객 조회 권한 확인
- 코멘트 또는 메시지 연결

## 데이터 구조 후보

### 접수 요청

```text
claim_id
customer_id
agent_id
status
title
description
created_at
submitted_at
completed_at
```

### 접수 파일

```text
claim_file_id
claim_id
file_id
file_type
created_at
```

### 접수 코멘트

```text
comment_id
claim_id
author_type
author_id
message
created_at
```

## 권한 확인 항목

- 고객이 본인 접수 내역만 볼 수 있는가
- 담당자가 연결된 고객의 접수만 볼 수 있는가
- 관리자가 전체 접수를 볼 수 있는가
- 파일 다운로드 권한이 접수 권한과 일치하는가
- 상태 변경 권한이 누구에게 있는가

## 주의사항

- 고객용 화면은 내부 관리자 정보가 노출되지 않게 한다.
- 고객이 제출한 파일은 업무 대상과 명확히 연결한다.
- 파일 삭제와 접수 상태 변경은 별도 권한으로 관리한다.
- 모바일에서 파일 선택 UX를 우선 고려한다.
