# 고객 청구 파일 접수 기능 자산

## 기능명

고객 청구 파일 접수 / Customer Claim File Intake

## 원본 프로젝트

- insurance

## 기능 목적

고객이 모바일 웹 또는 고객앱에서 필요한 파일을 직접 첨부하고, 담당자 또는 관리자가 이를 확인하여 청구, 접수, 검토, 처리 업무로 연결할 수 있게 하는 기능이다.

보험 프로젝트에서는 고객 청구 기능으로 시작하지만, 다른 프로젝트에서는 고객 서류 제출, 계약서류 제출, A/S 접수, 민원 접수, 검수 자료 제출 등으로 확장할 수 있다.

## 핵심 사용 시나리오

- 담당자가 고객에게 고객앱 링크를 전달한다.
- 고객은 링크로 접속해 본인 정보 또는 연결 코드를 통해 진입한다.
- 고객은 청구 또는 제출할 파일을 첨부한다.
- 파일은 스토리지에 저장되고 DB에는 접수 정보가 기록된다.
- 담당자는 관리자/업무 화면에서 접수 내역을 확인한다.
- 담당자는 상태를 변경하거나 추가 요청 메시지를 보낸다.

## 재사용 가능 분야

- 보험 청구 파일 접수
- 부동산 계약서류 제출
- 병원 진료/보험 서류 제출
- 학원 등록서류 제출
- B2B 거래 증빙자료 제출
- 에스크로 산출물 제출
- A/S 접수
- 민원/고객 요청 접수

## 자산 등급

- 등급 초안: B
- 이유: 고객앱, 고객 연결 코드, 파일관리, 메시지, 권한 구조가 함께 연결되므로 표준화가 필요하다.

## 기능 범위

### 포함

- 고객 접속 링크 또는 연결 코드
- 고객용 파일 제출 화면
- 파일 첨부
- 접수 내역 생성
- 담당자 확인 화면
- 접수 상태 관리
- 추가 요청 또는 코멘트 연결
- 파일관리 기능과 연결

### 제외

- 보험 청구 전용 필드 강제
- 특정 고객 테이블 고정
- 특정 스토리지 고정
- 특정 알림 채널 고정
- 특정 모바일 앱 구조 강제

## 표준 상태 후보

```text
requested
submitted
reviewing
need_more_info
completed
rejected
cancelled
```

## 표준화 원칙

- 고객앱 화면과 담당자 관리 화면을 분리한다.
- 파일 제출 기능은 storage-r2 기능 자산과 연결한다.
- 고객 연결은 별도 링크/토큰/코드 구조로 분리한다.
- 접수 상태는 프로젝트별 업무명에 맞게 확장 가능하게 둔다.
- 담당자 코멘트는 personal-message 기능과 연결 가능하게 설계한다.
- 고객이 볼 수 있는 정보와 내부 담당자가 볼 수 있는 정보를 분리한다.

## 투자/소개용 설명

고객이 별도 설치나 복잡한 절차 없이 모바일에서 파일을 제출하고, 담당자는 이를 업무 화면에서 바로 확인·처리할 수 있는 고객 서류 접수 자동화 기능이다.
