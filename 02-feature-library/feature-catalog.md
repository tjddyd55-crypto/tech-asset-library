# 기능 자산 카탈로그

프로젝트별 후보 기능 중 재사용 관점으로 선별한 통합 카탈로그다.

## 등급 기준

- `A`: 즉시 재사용 가능
- `B`: 경량 정리 후 재사용 가능
- `C`: 구조 재정비 필요
- `D`: 아이디어/참고용

- 기능명

- 필요한 프론트 구조

- 필요한 권한 구조

## 통합 목록 (초기 버전)

### 1) 멀티 롤 기반 메뉴/권한 매트릭스 (등급: A)

- 기능명: 멀티 롤 기반 메뉴/권한 매트릭스
- 원본 프로젝트: arkluk-erp
- 원본 코드 위치: `미기재 (추후 경로 등록)`
- 기능 목적: UI 노출/행위 권한을 역할 조합 기준으로 제어
- 재사용 가능 분야: 모든 관리자 시스템
- 필요한 프론트 구조: 라우트 가드, 컴포넌트 액션 가드
- 필요한 백엔드/API 구조: 권한 조회 API, 역할 관리 API
- 필요한 DB/스토리지 구조: RBAC 테이블 세트
- 필요한 권한 구조: 슈퍼관리자 + 감사 로그 필수
- 외부 의존성: SSO(선택)
- 재사용 난이도: 낮음
- 정리 필요 여부: 아니오
- 새 프로젝트 적용 지시문: 권한 키 네이밍 컨벤션을 먼저 확정 후 적용
- 투자/소개용 설명 문구: 확장형 플랫폼 운영의 보안/통제 기반을 제공

### 2) 타임슬롯 예약 충돌 방지 (등급: A)

- 기능명: 타임슬롯 예약 충돌 방지
- 원본 프로젝트: screengolf
- 원본 코드 위치: `미기재 (추후 경로 등록)`
- 기능 목적: 동시성 상황에서도 중복 예약을 차단
- 재사용 가능 분야: 예약 기반 서비스 전반
- 필요한 프론트 구조: 슬롯 선택 및 충돌 안내 UX
- 필요한 백엔드/API 구조: 트랜잭션 예약 API, 잠금 처리
- 필요한 DB/스토리지 구조: 슬롯 유니크 제약, 예약 레코드
- 필요한 권한 구조: 일반 사용자/운영자 대리 예약 권한
- 외부 의존성: 결제 API(선택)
- 재사용 난이도: 낮음
- 정리 필요 여부: 아니오
- 새 프로젝트 적용 지시문: 예약 트랜잭션 경계와 재시도 정책을 먼저 정의
- 투자/소개용 설명 문구: 사용자 신뢰를 높이는 예약 안정성 핵심 기능

### 3) 매물 상태 수명주기 관리 (등급: A)

- 기능명: 매물 상태 수명주기 관리
- 원본 프로젝트: global-estate
- 원본 코드 위치: `미기재 (추후 경로 등록)`
- 기능 목적: 도메인 객체 상태 전이를 정책 중심으로 통제
- 재사용 가능 분야: 상품/콘텐츠/공고 관리
- 필요한 프론트 구조: 상태 이력, 상태 전환 액션 UI
- 필요한 백엔드/API 구조: 상태 변경 API, 이력 API
- 필요한 DB/스토리지 구조: 본문 + 상태이력 분리
- 필요한 권한 구조: 작성자/검수자/운영자 분리
- 외부 의존성: 알림 채널(선택)
- 재사용 난이도: 낮음
- 정리 필요 여부: 아니오
- 새 프로젝트 적용 지시문: 상태 전이 규칙을 도메인 정책 객체로 분리
- 투자/소개용 설명 문구: 운영 일관성과 추적 가능성을 동시에 보장
