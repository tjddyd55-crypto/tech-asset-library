# 손사인 기능 자산

## 기능명

손사인 / 전자서명 기능

## 원본 프로젝트

- insurance

## 기능 목적

PC와 모바일에서 사용자가 직접 서명할 수 있도록 하고, 입력된 서명을 이미지 데이터로 변환하여 신청서, 동의서, 계약서, PDF 자동화, 청구 서류 등에 연결할 수 있게 하는 기능이다.

## 핵심 사용 시나리오

- 고객이 모바일에서 손가락으로 서명한다.
- 사용자가 PC에서 마우스로 서명한다.
- 작성된 서명을 이미지/base64/blob/file 형태로 저장한다.
- 저장된 서명을 PDF 문서 또는 신청서 이미지에 삽입한다.
- 저장 위치는 프로젝트마다 다르므로 기능 내부에 고정하지 않는다.

## 재사용 가능 분야

- 보험 신청서/동의서
- 부동산 계약서
- 병원 문진표/동의서
- 학원 등록서류
- B2B 계약/검수 확인서
- 에스크로 단계별 승인 확인
- 모바일 고객앱 서명 제출

## 자산 등급

- 등급 초안: A
- 이유: 비교적 독립 컴포넌트로 분리 가능하고, 여러 프로젝트에서 반복적으로 사용할 가능성이 높다.

## 기능 범위

### 포함

- 서명 입력 패드
- PC 마우스 입력
- 모바일 터치 입력
- 서명 지우기
- 서명 미리보기
- 서명 이미지 변환
- 저장 콜백 연결

### 제외

- 특정 DB 저장 로직
- 특정 파일 스토리지 업로드 로직
- 특정 PDF 템플릿 삽입 로직
- 특정 프로젝트의 고객/계약/신청서 ID 고정

## 표준화 원칙

- 서명 패드는 독립 UI 컴포넌트로 유지한다.
- 저장 위치는 외부에서 주입한다.
- 결과 데이터 형식은 최소 base64 또는 Blob으로 받을 수 있게 한다.
- PC/모바일 입력 이벤트를 모두 지원한다.
- 서명 완료 전 빈 서명 저장을 막는다.
- 서명 이미지는 배경 투명 PNG를 우선 고려한다.

## 새 프로젝트 적용 방식

1. UI 컴포넌트로 서명 패드를 추가한다.
2. 저장 버튼 클릭 시 서명 데이터를 부모 컴포넌트로 전달한다.
3. 부모/업무 화면에서 프로젝트별 저장 API를 호출한다.
4. PDF 자동화가 필요한 경우 서명 이미지를 PDF 삽입 모듈로 넘긴다.
5. 고객앱/모바일앱에서는 터치 이벤트와 화면 크기를 별도 확인한다.

## 투자/소개용 설명

사용자가 별도 장비 없이 PC 또는 모바일에서 직접 전자서명을 입력하고, 이를 계약서·신청서·동의서·청구서류 등에 연결할 수 있는 문서 자동화 기반 기능이다.
