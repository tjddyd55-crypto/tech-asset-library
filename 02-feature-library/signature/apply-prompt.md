# 손사인 기능 적용 지시문

```text
목표:
현재 프로젝트에 손사인/전자서명 기능을 적용한다.

기능 목적:
사용자가 PC 또는 모바일에서 직접 서명할 수 있고, 서명 결과를 이미지 데이터로 받아 신청서, 계약서, 동의서, 청구서류, PDF 자동화 등에 연결할 수 있게 한다.

적용 원칙:
1. 서명 패드 UI는 독립 컴포넌트로 만든다.
2. 저장 위치와 저장 API는 기능 내부에 고정하지 않는다.
3. 저장은 부모 화면 또는 업무 서비스에서 처리하도록 콜백 구조로 만든다.
4. PC 마우스 입력과 모바일 터치 입력을 모두 지원한다.
5. 빈 서명은 저장하지 못하게 한다.
6. 서명 결과는 base64 또는 Blob/File 형태로 받을 수 있게 한다.
7. PDF 삽입이 필요한 경우 PDF 좌표 자동화 기능과 연결 가능한 구조로 만든다.
8. PC와 모바일 화면이 다르면 화면 컴포넌트는 분리하되 서명 입력 로직은 공유한다.

필수 UX:
- 서명 영역
- 다시 쓰기 버튼
- 저장/완료 버튼
- 취소 버튼
- 저장 전 미리보기 또는 저장 완료 표시
- 모바일에서 손가락 입력 시 화면 스크롤과 충돌하지 않도록 처리

필수 오류 처리:
- 빈 서명 저장 방지
- 저장 실패 안내
- 네트워크 오류 안내
- 모바일 터치 입력 실패 방지
- 너무 작은 서명 영역 방지

데이터 처리:
- 기능 컴포넌트는 서명 데이터만 반환한다.
- ownerId, customerId, documentId, claimId 같은 프로젝트 고정값은 외부에서 전달받는다.
- DB 저장, R2/S3 업로드, PDF 삽입은 프로젝트별 서비스에서 처리한다.

절대 하지 말 것:
- 특정 프로젝트의 DB 테이블명을 서명 컴포넌트 내부에 하드코딩하지 말 것
- 특정 API 경로를 서명 컴포넌트 내부에 박지 말 것
- 모바일 전용 터치 이벤트만 구현하고 PC 마우스 입력을 누락하지 말 것
- PC 전용 마우스 이벤트만 구현하고 모바일 터치를 누락하지 말 것
- 기존 프로젝트 전체 코드를 통째로 복사하지 말 것

완료 후 보고:
- 생성/수정한 파일 목록
- 서명 데이터 반환 방식
- 저장 연결 방식
- PC/모바일 동작 방식
- 남은 TODO
```
