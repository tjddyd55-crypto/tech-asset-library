# 개인 메시지 기능 적용 지시문

```text
목표:
현재 프로젝트에 개인 메시지 기능을 적용한다.

기능 목적:
담당자, 관리자, 회사, 팀 또는 권한자가 특정 고객이나 사용자에게 개별 메시지를 보내고, 수신자가 메시지를 확인하거나 답변/코멘트를 남길 수 있게 한다.

적용 원칙:
1. 전체 공지와 개인 메시지를 분리한다.
2. 발신자와 수신자를 명확히 구분한다.
3. 고객용 화면과 내부 담당자 화면을 분리한다.
4. 메시지는 targetType, targetId로 업무 대상과 연결 가능하게 만든다.
5. 답변 또는 코멘트는 메시지 스레드 구조로 확장 가능하게 둔다.
6. 읽음 상태와 발송 상태를 별도 관리한다.
7. 첨부파일이 필요한 경우 storage-r2 기능 자산과 연결한다.
8. 알림이 필요한 경우 notification 기능과 연결 가능하게 둔다.

필수 발신자 UX:
- 수신자 선택
- 제목 입력
- 본문 입력
- 첨부파일 선택 옵션
- 발송
- 보낸 메시지 목록
- 메시지 상세/답변 확인

필수 수신자 UX:
- 받은 메시지 목록
- 메시지 상세
- 읽음 표시
- 답변 또는 코멘트 작성
- 첨부파일 확인

필수 서버 기능:
- 메시지 생성
- 수신자 연결
- 메시지 목록 조회
- 메시지 상세 조회
- 읽음 처리
- 답변/코멘트 등록
- 권한 확인

데이터 연결:
- senderType, senderId로 발신자를 구분한다.
- recipientType, recipientId로 수신자를 구분한다.
- targetType, targetId로 청구, 계약, 주문, 공고 등 업무 대상을 연결한다.
- fileId가 있는 경우 첨부파일로 연결한다.

절대 하지 말 것:
- 고객에게 내부 메모를 노출하지 말 것
- 전체 공지 기능과 개인 메시지 기능을 한 테이블/한 화면으로 강하게 묶지 말 것
- 수신자 권한 확인 없이 메시지를 노출하지 말 것
- 특정 프로젝트의 고객 ID 구조를 메시지 기능 내부에 고정하지 말 것
- PC용 메시지 관리 화면을 모바일 고객 화면에 그대로 사용하지 말 것

완료 후 보고:
- 생성/수정한 파일 목록
- 발신자 메시지 흐름
- 수신자 메시지 흐름
- 읽음/답변 처리 방식
- 업무 대상 연결 방식
- 남은 TODO
```
