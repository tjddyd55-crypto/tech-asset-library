# 개인 메시지 기능 자산

## 기능명

개인 메시지 / Personal Message

## 원본 프로젝트

- insurance

## 기능 목적

관리자, 담당자, 회사, 팀 또는 권한자가 특정 고객이나 사용자에게 개별 메시지를 보내고, 수신자는 고객앱 또는 사용자 화면에서 메시지를 확인하고 필요 시 답변이나 코멘트를 남길 수 있게 하는 기능이다.

## 핵심 사용 시나리오

- 담당자가 특정 고객에게 안내 메시지를 보낸다.
- 고객은 고객앱에서 메시지를 확인한다.
- 고객은 필요한 경우 답변 또는 코멘트를 남긴다.
- 담당자는 보낸 메시지와 고객 답변을 관리 화면에서 확인한다.
- 파일 접수, 청구, 계약, 일정, 공지 등 다른 업무와 메시지를 연결한다.

## 재사용 가능 분야

- 보험 고객 개별 안내
- 청구 보완 요청
- 부동산 계약 진행 안내
- 병원/학원 고객 안내
- B2B 주문 문의
- 에스크로 단계별 확인 요청
- 오디션 지원자 개별 안내
- 고객센터 1:1 문의

## 자산 등급

- 등급 초안: B
- 이유: 메시지 기능 자체는 범용성이 높지만 수신자 범위, 고객앱 연결, 알림, 파일/업무 연결 방식이 프로젝트마다 달라 분리가 필요하다.

## 기능 범위

### 포함

- 개인 메시지 작성
- 수신자 선택
- 메시지 발송
- 보낸 메시지 목록
- 받은 메시지 목록
- 메시지 상세
- 답변 또는 코멘트
- 읽음 상태
- 업무 대상 연결

### 제외

- 특정 고객앱 라우트 고정
- 특정 알림 채널 고정
- 특정 권한명 고정
- 특정 게시판 구조 강제
- 전체 공지 기능과의 강한 결합

## 표준화 원칙

- 메시지 작성자와 수신자를 명확히 구분한다.
- 메시지는 고객, 사용자, 팀, 회사 등 다양한 대상에 확장 가능해야 한다.
- 업무 연결 대상은 targetType, targetId로 분리한다.
- 답변/코멘트는 메시지 스레드 구조로 확장 가능하게 둔다.
- 고객에게 노출되는 메시지와 내부 메모를 분리한다.
- 읽음 상태와 발송 상태를 별도 관리한다.

```text

```

## 투자/소개용 설명

담당자와 고객 또는 내부 사용자 간의 개별 커뮤니케이션을 업무 흐름과 연결하여, 청구 보완 요청, 계약 안내, 파일 제출 요청, 개인 공지 등을 한 화면에서 관리할 수 있는 고객 커뮤니케이션 기반 기능이다.
