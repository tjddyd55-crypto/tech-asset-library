# PDF 좌표 자동화 기능 적용 지시문

```text
목표:
현재 프로젝트에 PDF 좌표 기반 자동 입력 기능을 적용한다.

기능 목적:
PDF 템플릿 위에 입력 좌표를 지정하고, 프로젝트의 업무 데이터를 해당 좌표에 자동 삽입하여 신청서, 계약서, 동의서, 청구서류 등을 자동 생성한다.

적용 원칙:
1. PDF 템플릿 관리와 좌표 필드 관리를 분리한다.
2. 좌표 필드는 특정 업종의 DB 컬럼명에 직접 고정하지 않는다.
3. fieldKey 기반으로 데이터를 매핑한다.
4. PDF 미리보기 화면과 최종 PDF 생성 로직을 분리한다.
5. 좌표값은 페이지 번호, x, y, width, height, fieldType을 함께 저장한다.
6. 텍스트, 날짜, 체크박스, 이미지, 서명 필드를 구분한다.
7. 서명 필드는 손사인 기능 자산과 연결 가능하게 만든다.
8. 템플릿 버전 변경 시 기존 좌표가 깨지지 않도록 버전 개념을 고려한다.

필수 UI:
- PDF 템플릿 업로드 또는 선택
- PDF 페이지 미리보기
- 좌표 찍기/수정/삭제
- 필드명/필드키 입력
- 필드 타입 선택
- 좌표 목록
- 테스트 데이터로 미리보기
- 최종 PDF 생성/다운로드

필수 API/서비스:
- PDF 템플릿 목록 조회
- PDF 템플릿 등록
- 좌표 필드 저장
- 좌표 필드 조회
- PDF 생성 요청
- 생성된 PDF 다운로드 또는 저장

데이터 매핑:
- 프로젝트별 업무 데이터는 별도 mapper에서 fieldKey 기준으로 변환한다.
- 예: customer.name, customer.birthDate, contract.date, signature.image
- PDF 생성 모듈은 원본 도메인 객체를 직접 알지 않도록 한다.

절대 하지 말 것:
- 특정 보험사/계약서/신청서 양식만을 기준으로 하드코딩하지 말 것
- PDF 좌표와 고객 DB 컬럼을 직접 강하게 결합하지 말 것
- 좌표 편집 UI와 PDF 생성 비즈니스 로직을 한 파일에 섞지 말 것
- 확대/축소 상태에서 좌표 보정 없이 저장하지 말 것
- 한글 폰트 문제를 무시하지 말 것

완료 후 보고:
- 생성/수정한 파일 목록
- PDF 템플릿/좌표/생성 흐름
- fieldKey 매핑 방식
- 서명 필드 연결 방식
- 남은 TODO
```
