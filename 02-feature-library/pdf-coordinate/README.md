# PDF 좌표 자동화 기능 자산

## 기능명

PDF 좌표 기반 자동 입력 / PDF Coordinate Automation

## 원본 프로젝트

- insurance

## 기능 목적

PDF 문서 위에 입력 좌표를 지정하고, 고객 정보·신청서 정보·서명 이미지·업무 데이터를 해당 좌표에 자동으로 삽입하여 신청서, 계약서, 동의서, 청구서류 등을 자동 생성하는 기능이다.

## 핵심 사용 시나리오

- 사용자가 PDF 신청서 템플릿을 등록한다.
- 관리자 또는 개발자가 PDF 위에 입력 좌표를 지정한다.
- 좌표마다 어떤 데이터가 들어갈지 매핑한다.
- 고객 정보, 계약 정보, 상담 정보, 서명 이미지 등을 자동 삽입한다.
- 완성된 PDF를 다운로드하거나 저장소에 보관한다.

## 재사용 가능 분야

- 보험 신청서 자동 작성
- 부동산 계약서 자동 작성
- 병원 문진표/동의서 자동 작성
- 학원 등록서류 자동 작성
- B2B 계약/발주/검수 문서 자동 작성
- 에스크로 단계별 확인서 자동 생성
- 각종 행정/민원 PDF 서류 자동화

## 자산 등급

- 등급 초안: B
- 이유: 기능 가치는 높지만 PDF 뷰어, 좌표 저장, 데이터 매핑, 파일 저장소, 권한 구조가 함께 연결되므로 표준화 작업이 필요하다.

## 기능 범위

### 포함

- PDF 템플릿 등록
- PDF 페이지 렌더링
- 좌표 찍기
- 좌표 목록 관리
- 필드명 매핑
- 텍스트 삽입
- 체크박스/선택값 삽입
- 서명 이미지 삽입
- 완성 PDF 생성
- 다운로드/저장 연결

### 제외

- 특정 보험사 양식 고정
- 특정 고객 DB 구조 고정
- 특정 스토리지 고정
- 특정 권한/라우트 고정
- 특정 업종 필드명 강제

## 표준화 원칙

- PDF 템플릿과 좌표 매핑은 분리한다.
- 좌표 데이터는 프로젝트별 도메인 필드와 직접 결합하지 않는다.
- 데이터 매핑은 외부에서 주입 가능한 구조로 둔다.
- PDF 생성 로직은 프론트 미리보기와 서버 최종 생성 중 프로젝트 상황에 맞게 선택한다.
- 좌표는 페이지 번호, x, y, width, height, fieldKey, fieldType으로 관리한다.
- 서명 이미지는 손사인 기능 자산과 연결 가능한 구조로 둔다.

## 권장 좌표 모델

```text
id
pdf_template_id
page_index
field_key
field_label
field_type
x
y
width
height
font_size
align
required
created_at
updated_at
```

## 필드 타입 후보

- text
- number
- date
- checkbox
- radio
- image
- signature
- multilineText

## 새 프로젝트 적용 방식

1. PDF 템플릿 관리 구조를 만든다.
2. 좌표 편집 UI를 만든다.
3. 좌표별 fieldKey를 등록한다.
4. 프로젝트별 데이터 매핑 함수를 만든다.
5. PDF 생성 모듈에서 좌표와 데이터를 결합한다.
6. 생성된 PDF를 다운로드 또는 저장소에 업로드한다.
7. 서명이 필요한 경우 signature 기능과 연결한다.

## 투자/소개용 설명

기존 종이/PDF 서류에 사람이 반복 입력하던 정보를 좌표 기반으로 자동 입력하여 신청서·계약서·동의서 작성 시간을 줄이고, 전자서명 및 파일관리와 연결할 수 있는 문서 자동화 핵심 기능이다.
