# R2/S3 파일관리 기능 리팩터링 메모

## 현재 목표

insurance와 arkluk-erp 프로젝트에서 사용한 파일관리 구조를 여러 프로젝트에서 재사용 가능한 표준 기능 자산으로 정리한다.

## 리팩터링 방향

### 1. 저장소 어댑터 분리

파일관리 기능은 특정 저장소에 고정되지 않아야 한다.

권장 구조:

```text
File UI -> File Service -> Storage Adapter -> Storage Provider
```

### 2. 메타데이터와 실제 파일 분리

DB에는 파일의 업무 연결 정보와 상태를 저장한다.

실제 파일은 외부 저장소에 보관한다.

### 3. 소유 범위와 업무 대상 분리

파일 소유 범위와 업무 연결 대상을 나눠야 한다.

- owner: 개인, 고객, 팀, 회사 등 파일 소유 범위
- target: 청구, 계약, 주문, 신청서 등 업무 대상

### 4. 용량 정책 분리

용량 제한은 프로젝트마다 다를 수 있으므로 별도 정책으로 둔다.

- 개인 기본 용량
- 팀 기본 용량
- 회사 기본 용량
- 관리자 조정 용량
- 추가 구매 용량

### 5. PC/모바일 UX 분리

PC에서는 테이블과 폴더 트리를 함께 사용할 수 있다.

모바일에서는 카드형 목록과 간단한 액션 중심으로 구성한다.

## 표준 모듈 후보

```text
FileUploader
FileList
FileTree
FilePreview
FileActions
StorageService
StorageAdapter
StorageQuotaService
```

## 검증 항목

- 파일 등록 가능
- 파일 목록 조회 가능
- 권한 없는 접근 차단
- 저장공간 사용량 계산 가능
- 관리자 용량 조정 가능
- 모바일 카드형 UI 사용 가능
- PC 폴더 트리 UI 사용 가능
- 삭제 후 목록과 사용량이 일치
