# R2/S3 파일관리 기능 자산

## 기능명

R2/S3 기반 파일관리 기능

## 원본 프로젝트

- insurance
- arkluk-erp

## 기능 목적

사용자가 업로드한 파일을 클라우드 오브젝트 스토리지에 저장하고, 프로젝트의 업무 데이터와 연결하여 고객 파일함, 팀 자료실, 주문 첨부파일, 청구 파일, 계약 서류 등을 관리하는 기능이다.

## 핵심 사용 시나리오

- 사용자가 업무 화면에서 파일을 첨부한다.
- 파일은 클라우드 스토리지에 저장된다.
- DB에는 파일명, 크기, 종류, 업무 연결 정보 같은 메타데이터를 저장한다.
- 사용자는 권한에 따라 파일을 조회, 다운로드, 삭제한다.
- 관리자는 개인, 팀, 회사 단위의 저장공간 사용량을 확인하고 조정한다.

## 재사용 가능 분야

- 보험 고객 파일함
- 고객 청구 파일 접수
- 팀 자료실
- 부동산 계약서류 관리
- B2B 주문 첨부파일
- 오디션 지원 자료 업로드
- 에스크로 산출물 업로드
- 병원/학원/프랜차이즈 서류 관리

## 자산 등급

- 등급 초안: B
- 이유: 기능 가치는 매우 높지만 권한, 용량, 삭제 정책, 업무 연결 방식이 프로젝트마다 달라 어댑터화가 필요하다.

## 기능 범위

### 포함

- 파일 업로드
- 파일 메타데이터 저장
- 파일 조회
- 파일 다운로드
- 파일 삭제
- 폴더 또는 업무 대상 연결
- 용량 사용량 계산
- 관리자 용량 조정
- 정리 작업

### 제외

- 특정 프로젝트의 고객, 팀, 주문 ID 고정
- 특정 스토리지 이름 고정
- 특정 권한명 고정
- 특정 화면 UI 강제

## 표준화 원칙

- 스토리지 제공자는 교체 가능하게 분리한다.
- 파일 메타데이터와 실제 파일 저장소를 분리한다.
- 파일 소유자와 업무 연결 대상을 명확히 나눈다.
- 삭제는 DB와 스토리지 정합성을 함께 고려한다.
- 업로드 실패와 메타데이터 저장 실패에 대한 정리 전략을 둔다.
- 용량 제한은 개인, 팀, 회사 단위로 확장 가능하게 설계한다.

## 투자/소개용 설명

고객, 팀, 거래, 계약, 주문 등 다양한 업무에서 발생하는 파일을 클라우드 스토리지에 저장하고, 권한·용량·삭제·업무 연결까지 관리할 수 있는 업무형 SaaS의 핵심 기반 기능이다.
