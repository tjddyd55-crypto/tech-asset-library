# R2/S3 파일관리 기능 적용 지시문

```text
목표:
현재 프로젝트에 클라우드 기반 파일관리 기능을 적용한다.

기능 목적:
사용자가 업무 화면에서 파일을 등록하고, 권한에 따라 조회/관리할 수 있게 한다.
파일은 클라우드 저장소에 두고, DB에는 파일 메타데이터와 업무 연결 정보를 저장한다.

적용 원칙:
1. 파일 저장소와 업무 도메인을 분리한다.
2. 파일 메타데이터는 DB에서 관리한다.
3. 실제 저장소 제공자는 교체 가능하게 만든다.
4. 개인, 고객, 팀, 회사 범위를 명확히 구분한다.
5. 권한 체크는 프론트와 서버 양쪽에서 처리한다.
6. 저장공간 사용량을 계산할 수 있게 한다.
7. 삭제와 정리 작업은 정합성을 고려한다.
8. 업로드 UI와 저장 로직을 분리한다.

필수 UI:
- 파일 선택
- 업로드 상태 표시
- 파일 목록
- 파일 상세 정보
- 다운로드 또는 열기 액션
- 삭제 확인
- 빈 상태 안내
- 오류 안내

필수 서버 기능:
- 파일 등록
- 파일 목록 조회
- 파일 접근 권한 확인
- 파일 메타데이터 저장
- 파일 관리 처리
- 저장공간 사용량 계산
- 관리자 설정 처리

데이터 연결:
- ownerType / ownerId로 소유 범위를 구분한다.
- targetType / targetId로 업무 대상을 연결한다.
- folderId가 있는 경우 폴더 구조와 연결한다.
- storageProvider와 storageKey로 실제 파일 위치를 추적한다.

절대 하지 말 것:
- 특정 프로젝트의 고객 ID나 팀 ID를 파일관리 내부에 고정하지 말 것
- 특정 저장소 이름을 모든 프로젝트의 표준으로 강제하지 말 것
- 파일 메타데이터 없이 실제 파일만 저장하지 말 것
- 권한 확인 없이 파일 접근을 허용하지 말 것
- PC용 파일 테이블을 모바일에 그대로 쓰지 말 것

완료 후 보고:

- 파일 등록 흐름
- 권한 확인 흐름
- 저장공간 계산 방식
```
