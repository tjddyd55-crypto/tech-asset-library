# 운영 로드맵

## Phase 1: 인벤토리 고정 (현재)

- 프로젝트별 기능 후보를 `01-project-inventory`에 정리
- 기능 등급 체계(A/B/C/D) 적용
- 코드 복사 없이 목적/구조/적용 조건 문서화

## Phase 2: 핵심 기능 자산화

- 등급 A/B 기능 우선으로 `02-feature-library` 상세화
- 공통 인터페이스/권한 모델/오류 처리 기준 확정
- 플랫폼별 차이점(웹/앱/설치형) 분리 가이드 작성

## Phase 3: 스타터킷 표준화

- `03-starter-kit` 기준으로 신규 프로젝트 킥오프 템플릿 운영
- 아키텍처/권한/디자인 시스템 규칙을 체크리스트화
- 프로젝트 초기 의사결정 기록 포맷 통일

## Phase 4: 대외 커뮤니케이션 패키지

- `05-showcase` 기반으로 기능 데모 스토리라인 정리
- 투자자/파트너 대상 기술 설명 템플릿 운영
- 기능 자산과 사업 가치 연결 문구 표준화
