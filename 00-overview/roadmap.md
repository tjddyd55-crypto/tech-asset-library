# 기술자산보관소 로드맵

## 0단계: 저장소 초기화

- README 작성
- 운영 원칙 정리
- 기본 폴더 구조 생성
- 프로젝트 목록 등록

## 1단계: 프로젝트 인벤토리

각 프로젝트별로 다음 항목을 정리한다.

- 프로젝트 목적
- 사용자 유형
- 권한 구조
- 주요 기능
- 재사용 가능한 기능 후보
- 스타터에 반영할 구조
- 우선 표준화할 기능

## 2단계: 기능 자산 카탈로그

기능 후보를 `02-feature-library/feature-catalog.md`에 등록한다.

- 기능명
- 분류
- 원본 프로젝트
- 재사용 등급
- 적용 난이도
- 의존성
- 정리 필요 여부

## 3단계: 핵심 기능 표준화

우선순위가 높은 기능부터 표준화한다.

1. 손사인
2. PDF 좌표 자동화
3. R2/S3 파일 업로드
4. 고객 청구 파일 접수
5. 개인 메시지
6. PC/모바일 분리 규칙
7. Electron 자동 업데이트
8. 폴더 트리 파일관리

## 4단계: 스타터킷 작성

새 프로젝트 시작 시 사용하는 기본 문서를 작성한다.

- 프로젝트 시작 체크리스트
- 아키텍처 규칙
- 플랫폼 분리 규칙
- 디자인 시스템 규칙
- 권한 구조 규칙
- 폴더 구조 템플릿

## 5단계: 프롬프트 템플릿 작성

Cursor/Codex에 바로 줄 수 있는 지시문을 만든다.

- 새 프로젝트 생성 지시문
- 기능 적용 지시문
- 기능 리팩터링 지시문
- 코드 표준화 지시문

## 6단계: 쇼케이스 정리

외부 설명용 문서를 만든다.

- 보유 기능 포트폴리오
- 투자자 설명 요약
- 데모 시나리오
- 적용 가능 산업군
