# 목적 정의

이 저장소는 "서비스 코드 저장소"가 아니라 "재사용 가능한 기술 자산 저장소"다.

## 왜 필요한가

- 프로젝트마다 반복 구현되는 핵심 기능을 자산화한다.
- 기능별 재사용 조건(프론트/백엔드/DB/권한)을 명시해 적용 비용을 낮춘다.
- 투자/사업 설명에 필요한 기술 포인트를 일관된 문구로 축적한다.

## 범위

- 포함: 기능 설명, 구조 요구사항, 적용 지시문, 데모/소개 문구
- 제외: 원본 프로젝트 전체 코드, 미정리 레거시 코드 덤프

## 성공 기준

- 신규 프로젝트 시작 시 `03-starter-kit`만으로 기본 구조를 합의할 수 있다.
- 기존 프로젝트 기능을 등급(A/B/C/D) 기반으로 빠르게 선별할 수 있다.
- 기능 문서만으로 구현 의도와 확장 포인트를 이해할 수 있다.
