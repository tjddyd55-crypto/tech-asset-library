# 기술자산보관소 목적

이 저장소는 여러 프로젝트에서 개발한 기능, 아키텍처 패턴, 디자인 규칙, 개발 지시문을 프로젝트 독립적인 자산으로 정리하기 위한 중앙 보관소다.

## 핵심 목적

- 새 프로젝트 시작 시 초기 아키텍처 설계 시간을 줄인다.
- 이미 개발한 기능을 다시 만들지 않고 재사용 가능한 형태로 정리한다.
- PC 웹, 모바일 웹, 모바일 앱, PC 설치형 프로그램 구조를 처음부터 분리해 설계한다.
- 기능별 원본 위치, 의존성, 권한, API/DB 구조를 기록한다.
- Cursor/Codex 같은 개발 에이전트에 바로 줄 수 있는 지시문을 축적한다.
- 투자자, 협력사, 외부 개발자에게 설명 가능한 기술 포트폴리오를 만든다.

## 운영 방향

이 저장소는 단순 코드 복사 저장소가 아니다.

기능을 보관할 때는 반드시 다음 정보를 함께 정리한다.

- 기능 목적
- 원본 프로젝트
- 원본 코드 위치
- 재사용 가능 범위
- 필요한 프론트 구조
- 필요한 백엔드/API 구조
- 필요한 DB/스토리지 구조
- 필요한 권한 구조
- 외부 의존성
- 새 프로젝트 적용 지시문

## 최종 목표

프로젝트 유형, 플랫폼, 권한, 기능, 디자인 방향을 선택하면 새 프로젝트 시작에 필요한 아키텍처 문서와 개발 지시문을 빠르게 생성할 수 있는 기술 자산 기반 스타터 시스템으로 확장한다.
