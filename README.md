# Tech Asset Library

여러 제품에서 검증된 기능, 아키텍처 패턴, 디자인 시스템 규칙, 개발 지시문, 투자/소개 자료를
프로젝트 독립적으로 축적하는 중앙 기술 자산 보관소입니다.

이 저장소의 목적은 단순 코드 보관이 아니라, 새 프로젝트를 시작할 때 필요한 기능 자산과 구조 규칙을 빠르게 선택하고 재사용할 수 있도록 정리하는 것입니다.

## 이 저장소로 만들어내는 것

- 신규 프로젝트 시작용 아키텍처 체크리스트
- PC 웹 / 모바일 웹 / 모바일 앱 / PC 설치형 프로그램 분리 기준
- 재사용 가능한 기능 자산 목록
- 기능별 원본 위치, 의존성, 적용 난이도 정리
- Cursor/Codex 개발 지시문
- 기능별 표준화/리팩터링 지시문
- 투자자/협력사 설명용 기능 포트폴리오

## 운영 원칙

- 실제 서비스 코드를 통째로 복사하지 않는다.
- 기능 설명 없는 소스 보관을 금지한다.
- 정리되지 않은 코드를 임의로 clean-version으로 저장하지 않는다.
- 특정 프로젝트 고정 구조(권한/라우트/스토리지)를 표준으로 강제하지 않는다.
- PC 웹, 모바일 웹, 앱, 설치형 클라이언트는 분리된 실행 환경으로 다룬다.
- 기능 내부에 저장 위치, 권한 정책, 라우트 구조를 고정하지 않는다.
- 코드보다 먼저 기능 목적, 사용 범위, 의존성, 적용 조건을 문서화한다.

## 저장소 구조

- `00-overview`: 목적, 자산 지도, 운영 로드맵
- `01-project-inventory`: 프로젝트별 기능 인벤토리 초안
- `02-feature-library`: 재사용 가능한 기능 자산 카탈로그
- `03-starter-kit`: 신규 프로젝트 시작 규칙/체크리스트
- `04-prompts`: 에이전트/개발 가이드용 프롬프트 템플릿
- `05-showcase`: 데모/투자자 커뮤니케이션용 문서

## 기능 등급 체계 (A/B/C/D)

- `A`: 즉시 재사용 가능. 구조/권한/오류 처리 문서화 완료.
- `B`: 소폭 정리 후 재사용 가능. 특정 프로젝트 결합 포인트 일부 존재.
- `C`: 개념/흐름은 유효하나 리팩터링 필요.
- `D`: 참고용 아이디어 단계. 직접 재사용 금지.

## 자산 등록 절차

1. `01-project-inventory`에 프로젝트별 기능 후보를 먼저 정리한다.
2. 후보 기능을 `02-feature-library/feature-catalog.md`에 등급 포함하여 등록한다.
3. 기능별 원본 프로젝트, 코드 위치, 의존성, 권한, API/DB 구조를 기록한다.
4. 등급 A/B 우선으로 적용 지시문과 투자/소개 문구를 정제한다.
5. 실제 코드 추출은 별도 승인 이후 최소 단위로 진행한다.
6. 정리된 코드는 원본 복사본이 아니라 재사용 가능한 표준 형태로 보관한다.

## 기능 자산 문서 기본 항목

각 기능은 아래 항목을 기준으로 정리합니다.

- 기능명
- 원본 프로젝트
- 원본 코드 위치
- 기능 목적
- 재사용 가능 분야
- 필요한 프론트 구조
- 필요한 백엔드/API 구조
- 필요한 DB/스토리지 구조
- 필요한 권한 구조
- 외부 의존성
- 재사용 난이도
- 정리 필요 여부
- 새 프로젝트 적용 지시문
- 투자/소개용 설명 문구

## 신규 프로젝트 시작 흐름

1. 프로젝트명, 업종, 대상 사용자, 플랫폼 범위를 정한다.
2. PC 웹 / 모바일 웹 / 모바일 앱 / PC 설치형 프로그램 필요 여부를 선택한다.
3. 필요한 권한 구조를 선택한다.
4. 사용할 기능 자산을 선택한다.
5. 디자인 시스템과 플랫폼 분리 규칙을 적용한다.
6. `04-prompts/new-project-setup-prompt.md` 기준으로 개발 지시문을 생성한다.
7. 기능은 `02-feature-library` 기준으로 가져오되, 새 프로젝트의 도메인/API/DB에 맞게 연결한다.

## 관리 대상 프로젝트

- `insurance`: 보험 업무 자동화
- `global-estate`: 지역 기반 자산거래 플랫폼
- `global_audition`: 글로벌 오디션/지원자 매칭 플랫폼
- `true-escrow`: 블록체인 기반 에스크로/증명 서비스
- `arkluk-erp`: B2B 거래 ERP
- `screengolf`: 스크린골프 데이터 수집/분석
